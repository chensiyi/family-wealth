# 行为体系：执行引擎与SOP

## 核心理念
情报系统负责“看”，行为体系负责“做”。在投资边界内，我们需要一套**机械化、纪律性、可验证**的执行引擎，以克服人性的贪婪与恐惧。这套体系不依赖主观情绪，而依赖预设的规则（Algorithm of Action）。

## 1. 决策机制：OODA 循环

将空战中的 OODA (Observe-Orient-Decide-Act) 引入资产管理：

1.  **观察 (Observe)**：接收情报系统的L3/L4输出（如：流动性枯竭信号）。
2.  **调整 (Orient)**：对比当前持仓与目标配置的偏差（如：权益类资产占比过高）。
3.  **决策 (Decide)**：生成交易指令（如：卖出10%纳指ETF，买入短期美债）。
4.  **行动 (Act)**：执行交易并确认结算。

## 2. 标准作业程序 (SOPs)

### SOP-01: 资金注入与建仓 (Capital Call)
*   **触发条件**：新增收入到账、资产价格进入低估区间（击球区）。
*   **步骤**：
    1.  核对资金来源合法性（合规）。
    2.  确认目标资产比例（配平）。
    3.  分批次下单（避免单一时点风险）。
    4.  记录成本价与理由。
*   **红线**：严禁在情绪高涨时一次性梭哈（All-in）。

### SOP-02: 利润回收与再平衡 (Rebalancing & Harvesting)
*   **触发条件**：
    *   *时间触发*：每季度/半年度末。
    *   *阈值触发*：某类资产偏离目标比例 ±10%（如：权益涨幅过大）。
*   **步骤**：
    1.  卖出超配资产（止盈）。
    2.  买入低配资产（补仓）。
    3.  提取部分利润进入“防御性现金池”（落袋为安）。
*   **意义**：自动高抛低吸，维持风险敞口恒定。

### SOP-03: 紧急熔断与撤退 (Emergency Break)
*   **触发条件**：情报系统发出“红色警报”（如：托管机构CDS飙升、战争爆发、账户冻结预警）。
*   **步骤**：
    1.  **一键止损**：不计成本变现流动性资产。
    2.  **资金转移**：启动备用转账通道，将资金移至安全法域/安全账户。
    3.  **物理备份**：打印关键凭证，下载数据备份。
*   **演练**：每年至少进行一次“桌面演练”。

## 3. 投资边界管理 (Boundary Control)

### 白名单制度 (Whitelist)
*   **允许投资**：
    *   发达国家主权债（高评级）。
    *   全球核心指数ETF（低费率、高流动性）。
    *   能看懂现金流逻辑的硬资产（如核心城市房产）。
    *   实物黄金（部分）。
*   **禁止投资（黑名单）**：
    *   看不懂底层逻辑的结构化产品（雪球、期权组合）。
    *   资金池业务（P2P、非标信托）。
    *   高杠杆衍生品。
    *   处于地缘政治高危区的资产。

### 仓位管理规则
*   **单一大类资产上限**：不超过总资产的 X%（如40%）。
*   **单一法域风险敞口**：根据家庭安全需求设定上限。
*   **杠杆率硬顶**：原则上不使用杠杆，或限制在极低水平（如LTV < 30%）。

## 4. 执行纪律与审计

*   **双人复核制**：大额转账或交易需家庭内部两人确认（Maker-Checker机制）。
*   **交易日志**：每一笔操作必须记录：时间、标的、价格、数量、决策依据（链接到情报系统）。
*   **结果偏离度分析**：定期对比实际收益与预期收益，检查执行过程是否有动作变形。

## 5. 工具箱
*   **交易指令单模板**：标准化填写，防止误操作。
*   **授权委托书**：提前签署的应急授权文件。
*   **硬件密钥**：U盾、冷钱包、2FA验证器的物理安全保管规范。
